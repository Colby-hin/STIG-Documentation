# STIG: WN11-CC-000325  
Automatically signing in the last interactive user after a system-initiated restart must be disabled

---

## Identifiers
- STIG ID: WN11-CC-000325
- Platform: Windows 11 Pro 64-bit (Azure VM)

---

## Finding

The system was configured to automatically sign in the last interactive user following a system-initiated restart.

This behavior allows cached credentials to be reused without user interaction.

---

## Security Impact

If automatic restart sign-on remains enabled:

- User credentials may be reused without reauthentication.
- Increased risk of session exposure.

---

## Remediation

Configured the system to disable Automatic Restart Sign-On via registry policy.

### PowerShell Remediation Script

```powershell
# Ensure the policy path exists
New-Item -Path "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" -Force | Out-Null

# Disable Automatic Restart Sign-On
Set-ItemProperty `
    -Path "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" `
    -Name "DisableAutomaticRestartSignOn" `
    -Value 1 `
    -Type DWord
```

---

## Validation

### PowerShell Validation Command

```powershell
Get-ItemProperty `
    -Path "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" `
    -Name "DisableAutomaticRestartSignOn"
```

Expected Result:

```
DisableAutomaticRestartSignOn : 1
```

---

## Operational Result

With DisableAutomaticRestartSignOn set to 1, the system no longer automatically logs in the last user after a restart.

Manual authentication is required following system reboot.
